# Data Policy

MetricDrive should be reproducible without shipping large or restricted data.

## Rules

- Do not commit raw dataset files.
- Do not commit private company data or non-public artifacts.
- Keep tiny synthetic fixtures in git only when they are hand-authored or
  generated by checked-in scripts.
- Keep large generated outputs under `outputs/`, `runs/`, or `artifacts/`.
- Document all public dataset download steps separately from the default demo.

## Local Layout

```text
data/raw/          public raw files downloaded locally
data/processed/    derived local files
outputs/           reports, figures, and experiment outputs
runs/              training runs and checkpoints
artifacts/         larger generated artifacts
```

Only `.gitkeep` placeholders are tracked under `data/`.
